# Auckland Church SEO Competitive Analysis

## Executive Summary

This document analyzes the search landscape for church-related queries in Auckland, New Zealand, examines top-ranking competitor websites, and provides actionable recommendations for ev.church to improve its Google search visibility. The research covers technical SEO, local SEO, content strategy, and structured data best practices specific to churches in 2026.

Key finding: ev.church has a strong foundation but is missing critical SEO elements -- structured data markup, a content/blog strategy, optimized Google Business Profiles per campus, and citation consistency -- that competitors like City Impact Church and C3 Church already leverage.

## 2. Competitor Website Analysis

### 2.1 City Impact Church (cityimpactchurch.com) -- BEST IN CLASS

**Strengths:**
- **Structured data:** Implements Organization, WebPage, BreadcrumbList, and ImageObject schema markup
- **Content strategy:** Sermon library with video, blog ("Church Blog"), Bible reading plans, devotionals, kids/youth content
- **Multi-campus:** 9 physical locations with dedicated location pages, each with unique NAP data
- **Local SEO:** Complete address, phone, and service times per location
- **SEO fundamentals:** Clear heading hierarchy (H1 > H2 > H3), keyword-rich headings
- **Title:** "Welcome to City Impact Church - There's a place for you"
- **Meta description:** Present and optimized

**Weaknesses:**
- Minor typo in title tag ("Theres" instead of "There's")
- Could add FAQ schema and Event schema for services

**Takeaway for ev.church:** City Impact is the benchmark for multi-campus church SEO in Auckland.

### 2.2 C3 Church (c3church.org.nz) -- STRONG LOCAL SEO

**Strengths:**
- **Excellent structured data:** Organization schema with legal name, address, contact, social profiles; LocalBusiness schema with physical address; WebSite schema
- **Multi-campus:** 5 NZ locations (Ellerslie, Manukau, Albany, West, Whangarei) + 3 international
- **Local SEO:** Complete address with lat/long coordinates (-36.9009972, 174.8127481), phone number, social links
- **Sermon content:** "Latest Message" section with YouTube integration, "Watch Online" link
- **Strong CTAs:** Connect Groups, Next Steps forms, event announcements

**Weaknesses:**
- No blog or editorial content strategy
- Limited "Our Stories" section without depth

### 2.3 LIFE Church (lifenz.org) -- GOOD UX, WEAK TECHNICAL SEO

**Strengths:**
- **Title:** "LIFE - A Church to Call Home" (compelling, branded)
- **Meta description:** "With campuses across Auckland and Tauranga, come experience a home where you can build faith, purpose, and community."
- **6 campuses** with dedicated location information
- **Content:** Discipleship groups, Night School, Alpha Course, water baptism, community impact, prayer, newsroom
- **Mobile-friendly:** Responsive design with Tailwind CSS

**Weaknesses:**
- **No structured data/schema markup at all** -- major missed opportunity
- **No sermon archive or blog** -- limits organic search potential
- Heading structure could be more keyword-optimized

### 2.4 Equippers Church (equipperschurch.com) -- BRAND-FOCUSED, SEO-WEAK

**Strengths:**
- Strong brand presence (music, conference, college)
- Multi-location with "Find a Church" functionality
- Newsroom section exists

**Weaknesses:**
- No visible meta tags or structured data
- No sermon archive
- Weak local SEO signals (no schema, no visible NAP per location)
- Brand-heavy keywords rather than search-intent keywords

### 2.5 Holy Trinity Cathedral (holy-trinity.org.nz) -- TECHNICALLY POOR

**Strengths:**
- Historic institution with inherent domain authority

**Weaknesses:**
- Built on Wix -- dynamically rendered content is poorly crawlable
- No visible structured data, heading structure, or local SEO signals
- No blog or content strategy apparent

## 4. Technical SEO Recommendations

### 4.1 Structured Data (Schema Markup) -- HIGHEST PRIORITY

Implement JSON-LD structured data on every page. The Church type (Thing > Place > CivicStructure > PlaceOfWorship > Church) is the most specific and appropriate schema type.

**Homepage schema -- Organization + Church for each campus:**

```json
{
  "@context": "https://schema.org",
  "@type": "Church",
  "name": "Ev Church",
  "alternateName": "Auckland Evangelical Church",
  "url": "https://ev.church",
  "logo": "https://ev.church/logo.png",
  "description": "A multi-campus evangelical church in Auckland, New Zealand with locations in Rosedale, Hillsborough, and Auckland Central.",
  "telephone": "+64-X-XXX-XXXX",
  "address": [
    {
      "@type": "PostalAddress",
      "streetAddress": "9-11 Rothwell Avenue",
      "addressLocality": "Rosedale",
      "addressRegion": "Auckland",
      "postalCode": "0632",
      "addressCountry": "NZ"
    }
  ],
  "geo": {
    "@type": "GeoCoordinates",
    "latitude": "-36.7275",
    "longitude": "174.7350"
  },
  "openingHoursSpecification": [
    {
      "@type": "OpeningHoursSpecification",
      "dayOfWeek": "Sunday",
      "opens": "10:15",
      "closes": "12:00"
    }
  ],
  "sameAs": [
    "https://www.facebook.com/aucklandev.co.nz/",
    "https://www.instagram.com/aucklandev.co.nz/"
  ]
}
```

**Each campus location page should have its own Church schema with unique address, geo coordinates, and opening hours.**

Additional schema types to implement:
- **Event schema** for Sunday services, Easter services, special events
- **VideoObject schema** for sermon videos
- **BreadcrumbList schema** for navigation
- **WebSite schema** with SearchAction for sitelinks search box
- **FAQ schema** on relevant pages (beliefs, visiting, etc.)

### 4.2 Meta Tags

Every page needs unique, keyword-optimized title tags and meta descriptions:

| Page | Title Tag (under 60 chars) | Meta Description (under 160 chars) |
|------|---------------------------|-------------------------------------|
| Home | Ev Church Auckland - Sunday Services in Rosedale, Hillsborough & Central | Join Ev Church for Sunday services at three Auckland locations. A welcoming community exploring faith in Jesus. Visit us at 10:15am or 5:15pm. |
| About | About Ev Church - Who We Are | Learn about Ev Church Auckland -- our vision, values, and story. We are one church in three locations across Auckland, NZ. |
| North Campus | Ev Church North - Rosedale, Auckland | Join Ev Church North at 9-11 Rothwell Ave, Rosedale. Sunday services at 10:15am. Kids and youth programs available. |
| Central Campus | Ev Church Central - Hillsborough, Auckland | Visit Ev Church Central at 80 Olsen Ave, Hillsborough. Sunday services at 10:15am and 5:15pm. All welcome. |
| Sermons | Sermons - Ev Church Auckland | Watch and listen to the latest sermons from Ev Church Auckland. Explore our sermon archive by series, topic, or speaker. |

### 4.3 Technical Fundamentals

- **Canonical tags** on every page to prevent duplicate content issues
- **Open Graph and Twitter Card meta tags** for social sharing
- **XML sitemap** submitted to Google Search Console (include location pages, sermon pages, blog posts)
- **robots.txt** properly configured
- **Image alt text** on all images (currently appears empty) -- describe the content meaningfully
- **Core Web Vitals** optimization (LCP, CLS, INP) -- JavaScript-heavy carousels (Swiper) may cause issues
- **301 redirect** from aucklandev.co.nz to ev.church (if not already in place) to consolidate link equity
- **HTTPS** everywhere (appears to be in place)
- **hreflang** not needed (single-language site)

## 5. Local SEO Strategy

### 5.1 Google Business Profile -- CRITICAL

Each campus MUST have its own Google Business Profile. This is the single most impactful action for local search visibility.

**Per-campus GBP requirements:**

| Field | North Campus | Central Campus | Unichurch |
|-------|-------------|----------------|-----------|
| Business Name | Ev Church North | Ev Church Central | Ev Church Unichurch |
| Category (Primary) | Church | Church | Church |
| Category (Secondary) | Evangelical church, Religious organization | Evangelical church, Religious organization | Evangelical church, Religious organization |
| Address | 9-11 Rothwell Ave, Rosedale | 80 Olsen Ave, Hillsborough | Old Government House, Auckland Central |
| Phone | Unique per campus | Unique per campus | Unique per campus |
| Website | ev.church/north | ev.church/central | ev.church/unichurch |
| Hours | Sunday 10:15am-12:00pm | Sunday 10:15am-12:00pm, 5:15pm-7:00pm | Sunday 5:15pm-7:00pm |
| Photos | Campus-specific photos | Campus-specific photos | Campus-specific photos |

**Critical rules for multi-campus GBP:**
- Each campus gets a **unique phone number** (do NOT share numbers)
- Each campus links to a **dedicated landing page**, not the homepage
- Post campus-specific updates weekly (events, sermon highlights, photos)
- Respond to ALL reviews within 24 hours
- Do NOT set a service area -- churches are destination businesses
- Add attributes: wheelchair accessible, parking, etc.

### 5.2 NZ Citation and Directory Listings

Submit ev.church to these directories with consistent NAP data for each campus:

**High Priority (DA 85+):**
- Google Business Profile (google.co.nz)
- Facebook Business Page (per campus)
- Apple Business Connect (apple.com)
- Bing Places (bing.com/maps)
- LinkedIn Company Page
- Instagram Business Profile

**Medium Priority (Church-Specific):**
- Walk on Water NZ (walknonwater.org.nz) -- major NZ church directory
- Auckland Church Network (aucklandchurchnetwork.nz)

**General NZ Directories:**

**NAP Consistency Rules:**
- Use EXACTLY the same business name format everywhere

- Recommendation: Primary brand "Ev Church" with "Auckland Evangelical Church" as a secondary/legal name
- Update the Yelp listing from "Auckland Evangelical Church" to "Ev Church" (or add "Ev Church" as the display name)

### 5.3 Reviews Strategy

- Actively request Google reviews from congregation members
- Create a short link (e.g., ev.church/review) that goes directly to the Google review form
- Aim for 50+ reviews per campus within 6 months
- Respond to every review (positive and negative) within 24 hours
- Mention specific campus details in review responses to reinforce local relevance

## 6. Content Strategy

### 6.1 Sermon Content -- Biggest Untapped Opportunity

Churches produce unique content every week (the sermon) but most fail to leverage it for SEO. ev.church should:

1. **Transcribe every sermon** -- each transcript becomes a keyword-rich page
2. **Create a searchable sermon archive** with filters by series, topic, speaker, date, and scripture reference
3. **Optimize sermon pages** with:
   - Title: "[Sermon Title] - [Series Name] | Ev Church Auckland"
   - H1: Sermon title
   - Summary paragraph (100-200 words)
   - Full transcript (expandable)
   - Embedded video/audio
   - VideoObject schema markup
   - Related sermons internal links
4. **Each sermon series** gets its own landing page with description, all sermons in the series, and relevant keywords

### 6.2 Blog Content -- Build Topical Authority

Launch a blog targeting these content clusters:

**Cluster 1: Local/Community (drives local traffic)**
- "Things to do in [Auckland suburb] this weekend"
- "Community events in Rosedale/Hillsborough"
- "Serving Auckland: How Ev Church gives back"

**Cluster 2: Faith and Life (drives organic traffic)**
- "What to expect at your first church visit"
- "Is church for me? A guide for newcomers"
- "What do evangelical Christians believe?"
- "How to read the Bible: A beginner's guide"
- "What is Easter and why do Christians celebrate it?"
- "Explaining Christianity" content (already a nav item -- expand it)

**Cluster 3: Ministry/Programs (drives specific queries)**
- "Youth groups in Auckland for teenagers"
- "Kids church programs Auckland"
- "Connect groups: What they are and how to join"
- "University Christian groups in Auckland"

**Publishing cadence:** Minimum 2 posts per month, ideally weekly.

### 6.3 Location Pages -- Essential for Multi-Campus SEO

Create dedicated, content-rich landing pages for each campus:

**URL structure:**
- ev.church/north (or ev.church/locations/rosedale)
- ev.church/central (or ev.church/locations/hillsborough)
- ev.church/unichurch (or ev.church/locations/auckland-central)

**Each page should include:**
- Campus-specific H1: "Ev Church North -- Rosedale, Auckland"
- Address with embedded Google Map
- Service times
- Parking and transport information
- Campus pastor/leader introduction
- Photos of the campus and community
- Upcoming campus-specific events
- "What to expect" section
- Campus-specific schema markup (Church type)
- Unique meta title and description

## 8. Schema.org Implementation Guide

### 8.1 Type Hierarchy for Churches

```
Thing > Place > CivicStructure > PlaceOfWorship > Church
```

The Church type inherits properties from all parent types and is the most specific type available for churches on Schema.org.

### 8.2 Essential Properties

| Property | Source | Example |
|----------|--------|---------|
| name | Thing | "Ev Church North" |
| description | Thing | "Evangelical church in Rosedale, Auckland" |
| url | Thing | "https://ev.church/north" |
| image | Thing | Campus photo URL |
| logo | Place | Ev Church logo URL |
| address | Place | PostalAddress object |
| geo | Place | GeoCoordinates (lat/long) |
| telephone | Place | "+64-9-XXX-XXXX" |
| openingHoursSpecification | CivicStructure | Sunday service times |
| sameAs | Thing | Social media profile URLs |
| aggregateRating | Place | Google review rating (if applicable) |
| hasMap | Place | Google Maps URL |
| isAccessibleForFree | Place | true |
| publicAccess | Place | true |

### 8.3 Additional Schema Types to Implement

- **Event** for each Sunday service, special events (Easter, Christmas)
- **VideoObject** for sermon videos
- **Article / BlogPosting** for blog content
- **BreadcrumbList** for navigation
- **WebSite** with SearchAction on homepage
- **FAQPage** on "What We Believe", "Plan Your Visit", and FAQ pages
